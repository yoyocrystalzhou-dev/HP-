# HP 专项 · 子世代 功能计划

> 目标：从「能对话的世界」变成「沉浸式霍格沃茨生活模拟器」。核心原则：**世界内置、生活自由、AI 负责生成真实感事件；只有需要客观裁决的地方才调用数值/检定**。

---

## 一、现状（已完成）

| 模块 | 状态 |
|---|---|
| 世界数据内置（人物卡 20 + 世界观硬规则） | ✅ |
| 三世代首页 + 角色创建向导（学院/血统/魔杖/擅长） | ✅ |
| 数值模型 + 初始值（学院/血统/魔杖/擅长生成）+ 数值面板可视化 | ✅ |
| 时间线引擎（原著锚点注入 + 自动推进，防跑偏） | ✅ |
| 对话引擎（AI 在笼子里叙事） | ✅（待真实 Key 验证） |

**关键缺口：现在的行动容易被理解成“固定菜单结算”。后续要把它改成“地点 / 意图 / 当前状态驱动的自由生活”，数值只在必要判定处介入。**

---

## 二、要做的功能（按优先级）

### ★ P-A 生活场景引擎 + 必要判定（最高优先 · 游戏的心脏）

让玩家不是在点菜单刷数值，而是在霍格沃茨生活。数值存在，但它像世界物理规则一样躲在幕后，只有当事情真的需要成败、风险、门槛时才出面。

- **判定引擎 `lib/checks.js`**：
  - `resolveCheck(stat, difficulty, {favored})` → `{tier, success, margin}`，tier=大成功/成功/失败/大失败
  - `examGrade(学术, favored)` → 原著等级 O/E/A/P/D/T
  - 受控随机（可调），高数值≈更稳；结果由程序定，AI 不得改判
- **生活场景入口（不是固定收益菜单）**：
  - 玩家选择 / 输入：地点、同行对象、意图、语气，例如「去图书馆」「和赫敏一起查尼可·勒梅」「在走廊闲逛」「去魁地奇球场看训练」。
  - 系统把地点、时间、当前原著节点、在场角色、玩家状态注入 prompt。
  - AI 根据这些信息生成生活片段、随机事件、人物互动、线索、误会、课堂插曲等。
  - 主控可以指定事件倾向：平静日常 / 遇见某人 / 触发支线 / 危险靠近 / 只逛不推进。
- **判定只在需要时触发**：
  - 施法、魔药、飞行、夜游躲避、课堂表现、考试、告白、被接纳、冲突胜负等才检定。
  - 「去图书馆」本身不等于读书成功或学术 +1；可能遇到赫敏、发现禁书区线索、撞见潘斯、只是安静地消磨下午，也可能由主控指定发生什么。
  - 检定结果作为 `【判定结果】` 注入 prompt，AI 依据结果叙事，但不把生活压扁成结算。

### ★ P-B 好感度系统（NPC 关系）

HP 互动的核心。

- 每个 NPC 一个好感度 0–100，4 阶段（陌生/朋友/亲密/恋人），存 `relationships[charId].favor`
- 社交行动 → 确定性 + 判定增减；好感度影响可触发的剧情、告白、结局
- 在数值面板展示「关系」列表
- 红线：好感度只能由程序结算，AI 不得自行宣布「你们成为好友/恋人」

### ★ P-B′ 原创角色（OC · 可攻略/可嫁）

让玩家加入自己的角色，并能攻略/结婚——**不破坏世界锁定**（原著锁死，OC 是玩家自有的附加数据）。

- 玩家创建 OC：名字 / 性别 / 性格 persona / 学院 / 与玩家的关系
- **OC 可与原著角色建立外围关系**（如「德拉科的发小」「纳威的表亲」）：创建时选 canon 角色 + 关系类型，注入世界数据；轮到 AI 演该 canon 角色时会按此关系对待 OC
- OC 作为世界成员注入，AI 按其 persona 扮演，与 canon 角色共处
- 好感度 + 结局对 OC 一视同仁（canon 与 OC 走同一套关系/婚姻逻辑）
- 红线：OC 是普通巫师，其关系只在**日常/社交层**生效，**不得改变原著大事件与角色既定命运**，不得凌驾原著设定（AI guard）
- 入口：角色创建后「添加身边的人」或人物界面 +（UI later）；近期可在自由 RP 引入，靠记忆系统记住
- 依赖：P-B 好感度 + P-H 结局

### ★ P-C 体力 / 日程压力（✅ 基础完成 · 需生活化）

不用固定「每周几次」死板菜单，改用**体力（0–100）+ 时间段 + 地点风险**控制生活节奏：
- 去不同地点 / 做不同事消耗不同体力，但不必每次都结算数值
- 体力不足 → 行动受阻（不掷骰、不结算，提示 /休息）
- `/休息`（睡觉）→ 体力全恢复，旁白演绎入睡醒来，时间顺势推进到次日
- 体力条醒目显示（低于 20 变红）；体力注入 prompt
- 后续补：上午 / 下午 / 晚上 / 深夜，不同地点在不同时段有不同事件池与风险

### P-D 专用名词卡 → 世界书（内容增强）

- `专用名词卡.txt` → `worldBook[]`（关键词触发注入）
- 聊到「黄油啤酒/魁地奇/古灵阁…」时自动补设定，丰富细节、省 token
- 纯内容工程，可随时插入

### P-E 考试成绩 + 学院分

- 期末考节点（时间线已有）→ 触发各科 examGrade；五/七年级 O.W.L./N.E.W.T.
- 学院分：课堂表现、帮助同学、违反校规、夜游被抓等由事件后果触发，程序结算，面板展示
- 成绩与学院分进入结局判定

### P-F 静默记忆与连续性

- ai-chat 引擎本有记忆提炼，kiosk 里已关「建议弹窗」→ 改为**静默后台**累积 worldMemory/storyMemory
- 保证长对话剧情前后一致、不失忆
- 记录生活痕迹：常去地点、常同行的人、被教授记住的表现、和 NPC 的共同经历、已经埋下的支线

### P-F′ AI 日常成长评估（已加 · 自由生活配套）

- 日常事件随机性太高，不能预先规定“地点 = 固定收益”。
- AI 可根据实际发生的具体事件，在回复末尾用结构化标签建议成长：`【日常成长：学术+1；亲和+1】`。
- 程序解析并应用，但限制幅度：普通事件通常 +1，明显有效 +2，强烈事件最多 +3；学院分和体力有单独上限。
- 没有明确成长就不写，避免每轮都刷数值。

### P-F″ 时间段 / 日程氛围（已加 · 自由生活配套）

- 在输入区增加生活时间段：上午 / 下午 / 晚饭后 / 夜晚 / 深夜。
- 时间段只影响 AI 的事件池和生活氛围，不代表固定剧情或固定收益。
- 深夜会提示宵禁风险，下午更适合课程后活动，晚饭后更适合礼堂 / 公共休息室 / 走廊闲聊。
- `/休息` 会把时间段恢复到上午。

### P-G 存档 / 新游戏

- 现在每个世代单存档（自动续玩）
- 补：**重新开始 / 重置存档**；可选多存档槽

### P-H 结局系统（AI 多元生成，不写死）

- **结局不查表、不固定**——由 AI 依据七年的终值（数值/好感度/学院分/关系）**生成**，千人千面，保留 AI 多元性。
- 终值只作为「素材/倾向」喂给 AI（如高魔法值倾向傲罗、好感度高倾向与某角色结缘），不强制对应某个固定结局。
- 十九年后 epilogue 同理，开放生成。

### ★ 数值/好感度门槛裁决（已加 · 横切规则）

> 核心：**自由叙述不能空口兑现需要数值/好感度的结果**。AI 作为旁白可以自由生成生活，但必须尊重真实数字卡。

- 注入玩家当前数值 + 好感度 + 裁决规则（`lib/stats.js` `GATING_RULES`）
- 凡涉及「入选魁地奇队 / 施法成功 / 考试成绩 / 成为恋人 / 被接纳」等，旁白必须按数值裁决；不足则受挫/被拒，不顺着玩家实现
- 关键成败和好感度变化由系统结算；无法预设的日常成长可由 AI 评估后交给系统解析和限幅
- 例：好感度＜60 不能在一起；敏捷低难入队；学术低考试平平

## 三、推荐推进顺序

```
P-A 生活场景入口 + 必要判定
   →  地点/时间段事件池  →  P-B 好感度
   →  P-C 体力/日程压力  →  P-F 静默记忆
   →  P-E 考试/学院分  →  P-G 存档  →  P-H 结局
```

**里程碑**：做完 P-A 就该接 API Key 实测——重点验证「自由生活是否成立」「AI 是否会主动生成有趣事件」「检定是否只在该出现时出现」。

## 四、已定设计点

> 定调：**这是一个沉浸式霍格沃茨生活模拟器，不是跑团模拟器。** 检定逻辑只作为部分判定的底层工具，不是玩法外壳。

1. **入口方式**：✅ 两者都要——**地点/场景按钮 + 自由输入**。
   - 地点按钮只表示「我想去哪里 / 做什么倾向」，不是固定收益菜单。
   - 输入框仍可完全自由，例如「我想翘掉魔法史，偷偷跟着双胞胎去看他们在搞什么」。
2. **主控控制**：✅ 需要支持主控指定事件。
   - 例如「去图书馆，但让德拉科出现」「今晚不要触发大事件，只想日常」「触发一点危险线索」。
3. **随机事件**：✅ 地点 + 时间 + 人物 + 好感度 + 原著节点共同决定事件池。
   - 图书馆可能是安静阅读、遇见赫敏、禁书区线索、被平斯夫人警告、撞见某人传纸条，不是固定 `学术 +1`。
4. **判定透明度**：可调。
   - 重要判定可以显示骰点/数值。
   - 日常小判定可以弱化显示，只让 AI 描写结果，避免破坏沉浸。
5. **养成节奏**：待定（倾向时间段 + 体力 + 校历，而不是每周固定次数）。
6. **失败惩罚**：待定（倾向轻量：扣学院分 / 轻微受伤 / 好感度降，不致命）。

### 检定逻辑补充

- 不是每次行动都检定。只有当场景中出现明确成败、风险、门槛、关系推进时，app 才掷骰 / 结算。
- 检定结果只裁决那一件事，不替代 AI 的场景生成。
- AI 可以自由生成随机事件、NPC 反应、环境变化、支线线索；但不能改写程序已经裁决的成败和数值变化。
