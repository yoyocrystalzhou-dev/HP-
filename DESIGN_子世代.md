# HP 专项 · 子世代 设计方案

> 目标：把通用的 ai-chat（自由输入、AI 不可控）收窄成「哈利波特专项」，三世代各为一个**预先锁定好的世界**。本方案先完整设计**子世代**。

## 世代定义（锚点 = 哈利）

| 世代 | 是谁 | 时间线 |
|---|---|---|
| 祖世代 | 哈利的祖父母辈 | 更早 |
| 亲世代 | 哈利的父母辈（James/Lily，劫盗者时代） | ~1971 起 |
| **子世代（本方案）** | **哈利、罗恩、赫敏、德拉科…这一代学生** | **1991 年 9 月起** |

> ✅ 用户已确认：**子世代 = 哈利那一代（1991）**。因此现有 6 个文件（总体世界观 / 人物群像卡 / 专用名词卡 / 编年时间线 / 校历 / 原作剧情卡）**就是子世代的内容**，直接转换即可，无需新写。
> ✅ 工程形态：**fork 成独立的「HP 专项」仓库**。

---

## 0. 结论先行

**HP 专项 = fork 现有 ai-chat 引擎 + 「世代预设包」+「受限角色创建向导」+「自由生活场景层」。不重写引擎。**

理由：ai-chat 已经是一套成熟的「世界观锁定」引擎，旧痛点（输入太多、AI 乱跑）正好被它已有的能力 + 我们新增的「预设包 + 硬约束」解决。但产品定调不是跑团模拟器，而是**沉浸式霍格沃茨生活模拟器**：AI 的自由生成是核心，数值 / 检定只是部分场景的底层裁决。

## 1. 为什么能直接复用 ai-chat（能力盘点）

ai-chat 的核心数据单位是 **Project = 一个完整世界**，已包含：

| ai-chat 已有能力 | 对应「锁定世界观」的需求 |
|---|---|
| `instructions`（项目指令） | 世界观硬规则、写作风格、红线 |
| `worldBook[]`（关键词触发注入） | 专用名词卡：聊到才注入，省 token |
| `files[]`（项目文件） | 时间线、校历、剧情大纲等长文档 |
| `worldMemory`（客观事实） | 不可违反的设定事实 |
| `storyMemory`（主线时间线，带 date） | 编年史 / 原作剧情进度 |
| `characters[]`（人设 + 状态 + 关系 + 经历） | 预制 NPC 角色卡 |
| `currentState` + `forbiddenAssumptions` | **当前场景锁定 + 反编造硬约束** |
| 时间系统 `currentTimeLabel` | 学年/学期推进 |
| `player`（玩家角色） | 用户扮演的新生 |
| `importChars.js` TXT 批量导入 | **人物群像卡.txt 可直接导入**（格式已兼容） |
| 多模型流式（Anthropic/DeepSeek/OpenAI）、localStorage | 不动 |

**关键约束机制（已存在，直接复用）**：`buildSystem()` 的注入顺序固定为
`instructions → 项目文件 → 世界书 → 当前时间 → 当前剧情状态 → 严格约束(forbidden) → 客观事实 → 主线进度 → 角色 → 玩家 → 反编造 guardrails`。
其中已经内置了「禁止编造玩家身份/关系/经历」「未建立关系必须表现为初次见面」等硬规则——这正是我们要的「把 AI 关进笼子」。

## 2. 核心新增概念：世代预设包（Generation Preset）

一个**内置在代码里的 Project 种子常量**（不是用户上传、不暴露成可编辑面板）。三世代 = 三个内置预设包。结构就是标准 Project，外加锁定标记：

```jsonc
// presets/childGen.js —— 内置常量，玩家看不到、不可编辑
{
  "presetId": "child_gen",
  "name": "子世代 · 1991 霍格沃茨",
  "locked": true,                       // 全部世界数据只读，UI 不放出来
  "era": { "start": "1991年9月", "tone": "校园日常 / 成长 / 友谊 / 养成" },
  "instructions": "...世界观硬规则 + 写作风格 + 红线...",
  "files": [ /* 校历、时间线、剧情大纲 */ ],
  "worldBook": [ /* 专用名词卡，每条 {title, keywords[], content} */ ],
  "worldMemory": [ /* 不可违反的客观事实 */ ],
  "storyMemory": [ /* 编年时间线 StoryEvent[] */ ],
  "characters": [ /* 预制 NPC 角色卡：哈利/罗恩/赫敏/德拉科… */ ],
  "currentTimeLabel": "1991年9月1日 · 开学",
  "currentState": { "location": "霍格沃茨特快列车", "forbiddenAssumptions": [...] },
  "player": null                        // 由角色创建向导填入
}
```

### 文件 → ai-chat 字段 映射表（你给的素材怎么用）

| 你的文件 | 转成 | 做法 |
|---|---|---|
| 总体世界观.txt | `instructions` + 1~2 个 `files` | 硬规则进 instructions，长设定进项目文件 |
| 专用名词卡.txt | `worldBook[]` | 每个名词 = 一条 entry，title=名词，keywords=名词+别名 |
| 人物群像卡.txt | `characters[]` | **直接用现成 TXT 导入**（格式已兼容，无需改） |
| 哈利波特编年时间线.docx | `storyMemory[]` | 每条带 date 的 StoryEvent |
| 霍格沃兹校历.docx | 时间系统 + 1 个 `file` | 学期/节日驱动 `currentTimeLabel` 推进 |
| 原作剧情卡.txt | `storyMemory` 或 `file` | 主线进度参照 |
| 纯享版.txt | 写作风格 `file`（范文） | 作为文风样本注入 |

> ✅ 这 6 个文件就是**子世代（1991，哈利那一代）**的素材，直接转换成预设包即可。祖世代 / 亲世代将来照同样格式各产一套。

## 3. 子世代世界观锁定（1991，哈利那一代）

- **时间线**：1991 年 9 月 1 日开学起（玩家与哈利同届入学）。「后伏地魔时代」的虚假和平——表面平静、暗流涌动，是后续七年剧情的核心张力。基调=**校园日常 / 成长 / 友谊 / 养成**（见素材里的休闲化处理）。
- **锁定 NPC（直接来自人物群像卡.txt，批量导入）**：
  哈利、罗恩、赫敏、纳威、德拉科、潘西、塞德里克、汉娜、秋·张、卢娜、金妮、韦斯莱双胞胎… + 教授（邓布利多、麦格、斯内普、弗立维、斯普劳特、海格…）。
- **硬规则（写进 `instructions` + `forbiddenAssumptions`，多数素材已写好，直接抽取）**：
  - 时间锁定在 1991 学年线上，不得跳到原著之后的剧情或提前剧透。
  - 伏地魔此时是逃逸的灵魂/暗线，**不得让其在日常场景中实体复活**；三大不可饶恕咒仅作概念，不触发使用。
  - 四学院体系、魁地奇规则、O.W.L./N.E.W.T.、货币体系（1 加隆=17 西可=493 纳特）固定不变。
  - 角色性格/口头禅/血统/关系 100% 遵循人物群像卡，不得 OOC、不得编造未给出的玩家信息。
  - 养成/好感度/学院分等数值玩法按专用名词卡 & 总体世界观的设定执行。

## 4. 受限角色创建向导（唯一需要新写的前端）

用**选择题**替代**自由填空**，是控制随机性的核心。6~7 步 → 生成 `player` + 初始化 `currentState`：

1. 身份：新生 / 转校生
2. 学院：选择 or 分院帽测试（4 选 1）
3. 血统：纯血 / 混血 / 麻瓜出身（影响初始好感度与剧情线索）
4. 年级 / 入学年份
5. 魔杖：木材 + 杖芯 + 长度（下拉）
6. 特长：1~2 个学科标签
7. 个性：3 个标签 + **一句**自我介绍（唯一自由文本，限字数）

输出写入 ai-chat 既有的 `player.persona` / `player.state`，其余引擎逻辑零改动。

学院与魔杖不是纯装饰：
- 每个学院显示说明与初始属性加成（如格兰芬多偏勇气、拉文克劳偏学术）。
- 每种魔杖木材 / 杖芯显示解释与属性加成，创建时计入初始数值。
- 这些只影响起点倾向，不锁死玩法路线。

## 5. 架构原则：引擎留后台，管理 UI 全砍，世界内置

> 核心：HP 专项是**成品**不是工具。ai-chat 把一切都「放出来」给用户编辑——HP 专项**反过来，世界写死藏在后台，玩家只看见玩的部分**。

**分三层：**

```
玩家 UI（新写，极简）
   世代选择 → 角色创建向导 → 对话 + 地点/场景入口 + 数值/好感度状态条
        │
引擎（保留 ai-chat 后台逻辑 + 新增数值层，不出现在界面上）
   buildSystem 注入 · callAPI 流式 · 生活事件生成 · 记忆/状态更新 · 时间系统 · 存档
   ＋ 数值系统 stats.js · 判定引擎 checks.js（只裁决必要的成败/门槛）
        │
世界数据（内置常量 presets/childGen.js，只读，玩家看不到）
   instructions · worldBook · characters · worldMemory · storyMemory · 校历
```

**砍掉 / 不放出来的 ai-chat 面板**（它们是给「造世界的人」用的，玩家用不到）：
`WorldBookPanel`、`CharacterPanel` 的编辑、`ProjectFilesPanel`、`CurrentStatePanel` 编辑、`ProjectListView`、`ProjectSettingsPanel`、`PendingUpdatesPanel`（记忆更新转为**静默后台**执行，不弹给玩家确认）。

**保留 / 新写的玩家界面：**
- `GenerationSelect.jsx`（首页三世代选择）
- `CharacterCreator.jsx`（角色创建向导，6~7 步选择题 → 写入 `player`）
- 对话主界面（**精简版**：只留消息流 + 输入框 + 每条消息的 Regenerate/Copy/Edit/Delete）
- `StatusBar`（玩家自己的养成数值/好感度/学院分，**只读展示**，非编辑器）
- 极简设置（仅 API key/模型，藏在角落）

**实例化方式：** 进入子世代 = `loadPreset(childGen)` 在后台把内置常量实例化成一个隐藏的 Project 喂给引擎，玩家无感。世界数据不写进 localStorage 的可编辑层（避免被改/被导出），只有 player 和对话/存档落盘。

## 6. 自由生活场景 + 必要判定

> 原则：**玩家是在霍格沃茨生活，不是在点行动菜单刷数值。** AI 负责自由生成生活片段、随机事件、NPC 互动与支线；程序只在需要客观裁决的地方做判定，防止 AI 随口兑现不合理结果。

### 6.1 数值清单（来自专用名词卡，内置）

全局养成数值（存 `player.stats`，从设定初值累积）：
**学术值 · 魔法值 · 勇气值 · 亲和力 · 敏捷值 · 亲情值**，外加 **学院分**（影响学院杯）。
每个 NPC 一个 **好感度**（0–100，分 陌生/朋友/亲密朋友/恋人 四阶段，存在 `relationships[charId].favor`）。
玩家创建时选的 **擅长学科 / 擅长魔咒** 给对应判定加成。

### 6.2 生活场景入口（地点 / 意图 / 主控）

玩家的入口不应是固定收益菜单，而是“我想去哪里、和谁、怀着什么意图”：

```text
去图书馆
和赫敏一起查资料
去魁地奇球场看训练
晚饭后在走廊闲逛
翘掉魔法史，偷偷跟着韦斯莱双胞胎
```

系统把以下信息注入 prompt：
- 当前时间段 / 日期 / 原著节点
- 地点与可能在场人物
- 玩家意图、同行对象、体力、好感度、近期记忆
- 主控指定的事件倾向（如平静日常 / 遇见某人 / 触发支线 / 危险靠近 / 暂不推进大事件）

AI 根据这些信息生成具体生活内容。比如“去图书馆”不等于简单读书并 `学术 +1`，它可以发展成：
- 平静读书，补足某门课的理解；
- 遇见赫敏，被她纠正一个错误；
- 被平斯夫人警告；
- 在书架间听见潘西与德拉科的低声谈话；
- 发现禁书区相关线索；
- 什么大事都没发生，只留下一个安静下午。

### 6.3 判定引擎（`lib/checks.js`，app 算，AI 不算）

只有「有明确成败 / 风险 / 门槛」的时刻才由程序结算，公式透明可调：

```
score = 相关数值 + 擅长加成 + 受控随机(roll) − 难度
tier  = 大成功 (score ≥ +15) / 成功 (≥ 0) / 失败 (< 0) / 大失败 (≤ −15)
```

- **施法/魔药/飞行/夜游躲避/课堂表现**：相关数值 + 擅长 + 随机。
- **期末成绩**：`grade = 阶梯(累积学术值 + 擅长)` → 原著等级 **O / E / A / P / D / T**（优秀→巨魔）。
- **关系门槛**：告白、被信任、被邀请加入小圈子等看好感度与事件积累。
- **受控随机**：用可固定的种子，保证可复现、可调，不是 AI 的随机。

结果作为 `【本回合判定结果：漂浮咒 → 成功（魔法值30 / 难度25）】` 注入 prompt，并配硬规则：
**「你必须依据【判定结果】叙事；禁止自行决定成败、分数或数值增减。」**

### 6.4 数值变化（来自事件后果，而不是地点菜单）

- 不把地点固定成收益表：去图书馆不自动等于 `学术 +1`，去球场不自动等于 `敏捷 +1`。
- 数值变化来自场景内发生的具体事件：完成一次有效练习、答对教授提问、与同学合作成功、夜游被抓、危机中帮助某人等。
- AI 可以生成事件内容，但具体数值增减仍由 app 依据事件类型 / 判定结果结算，防止乱加。
- 因为日常事件高度随机，app 不预设所有收益表。AI 可以根据本轮实际发生的事件自由评估小幅成长，并用 `【日常成长：学术+1；亲和+1】` 交给系统解析、限幅和应用。

### 6.5 触发模型（推荐：自由生活 + 必要裁决）

- **自由生活** = 玩家自然描述想做的事，AI 生成场景、事件和人物互动。
- **必要裁决** = 当场景中出现施法、考试、告白、被抓、冲突、入选、信任门槛等，app 判定后把结果交给 AI 演绎。
- **主控介入** = 玩家可指定“这次想发生什么倾向”，但不需要把一切压成按钮。
- 好处：AI 的自由性保留，世界规则也不会失控。

### 6.6 可视化（玩家界面）

- **StatusBar**：6 条养成数值条 + 学院分 + 好感度列表（只读展示）。
- **结算提示**：重要判定后弹出「漂浮咒 成功」「学院分 -5」「赫敏好感度 +3」之类的反馈。
- **沉浸优先**：日常小变化可以弱提示或合并到日终总结，避免每一步都像数值游戏。
- 数值变化即时反映到状态条。

## 7. 时间驱动的原著剧情引擎（防跑偏核心）

> 原则：**时间由程序随剧情主动推进；到达原著节点时自动注入对应原著剧情**，让故事始终贴着 1991–1998 主线，不跑偏。时间对玩家**可见**（聊天头部显示「学年阶段 · 日期」），但**推进由引擎控制，不由 AI 决定**。

### 7.1 原著节点表（内置 `canonTimeline`）

把 **编年时间线 + 校历 + 原作剧情卡** 转成一条有序节点表：

```jsonc
[
  { id, date: "1991-09-01", phase: "第一学年", title: "开学 · 分院",
    canon: "新生乘特快列车抵校，分院仪式，开学宴",
    inject: "本阶段应自然发生：分院、初识同学、熟悉城堡",
    weight: "routine" },
  { id, date: "1991-10-31", title: "万圣节巨怪事件",
    canon: "巨怪闯入城堡，赫敏被困女生盥洗室，哈利与罗恩出手相救，三人结为好友",
    inject: "让该事件在玩家周围自然发生；玩家可参与或旁观，但事件本身必须发生",
    weight: "major" },
  // …圣诞、期末考、学年末、魔法石高潮…七年逐年铺
]
```

节点分两类：**校历常规**（开学/周末霍格莫德/圣诞/期末考/学年末）+ **原著大事件**（巨怪/魔法石…）。

### 7.2 时间推进（`lib/timeline.js`，app 控制）

不被动等对话里出现时间，而是引擎按剧情进度主动推进：
- 场景收束 / 累计 N 轮 / 养成与校历活动（上一周课、过周末）→ 推进到下一日或下一节点；
- 玩家也可主动「推进到下一阶段」；
- 推进后更新 `currentTimeLabel`（玩家可见），并重算「当前所处节点」。

### 7.3 原著触发（anti-跑偏）

当时间到达 / 跨过某节点：
- 把该节点的 `canon` + `inject` 作为 `【当前阶段 · 原著剧情锚点】` 注入 prompt（位于硬规则之下、当前状态之上）；
- 要求 AI：**在玩家自由行动的同时，让该原著事件在背景/周围自然发生，保持与原著一致**；
- 同时把已发生节点写入 `storyMemory`，避免重复与前后矛盾。

### 7.4 与数值系统联动

期末考节点自动触发「期末成绩」判定（用累积学术值算 O/E/A/P/D/T）；大事件节点可挂勾相应判定（如对抗类事件看勇气/魔法值）。

### 7.5 推进节奏（待定，见第 9 节决策）

- **A 场景驱动**：每个场景收束就往前走一小步（自然沉浸）。
- **B 节点驱动**：停在当前节点，玩家「推进」才跳下一个（最可控、最贴原著）。
- **C 混合（推荐）**：日常用场景驱动小步走，大事件用节点强锚定。

## 8. 落地步骤 / MVP 拆分

遵守 ai-chat 的开发铁律（小步、每步 `npx vite build` 通过、不自动 commit/push、最小改动）：

- **P0 Fork**：把 ai-chat 复制成独立的「HP 专项」仓库，跑起来确认基线可用。
- **P1 子世代预设包（最大工作量）**：把 6 个文件转成 `presets/childGen.js`——
  - 人物群像卡 → `characters[]`（直接走 TXT 导入管线）
  - 专用名词卡 → `worldBook[]`（逐条 title+keywords+content）
  - 总体世界观 → `instructions` + 项目文件 + 硬规则
  - 编年时间线 / 原作剧情卡 → `storyMemory[]`
